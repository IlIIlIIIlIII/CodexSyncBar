# 포커스 해제 시 일회성 알림 제거 설계

## 목표

`계정 별칭을 저장했습니다.`처럼 사용자가 한 번 확인하면 충분한 성공 알림은 처음 열린 화면에서 표시한다. 해당 팝오버나 설정창을 닫거나 앱이 포커스를 잃은 뒤에는 제거하여 다음에 화면을 열 때 다시 나타나지 않게 한다.

오류와 경고처럼 후속 조치가 필요한 알림은 기존처럼 유지한다.

## 선택한 방식

일회성 알림을 일반 `banner`와 구분해 현재 알림의 ID를 모델이 추적한다. 기존 `showTransientBanner`가 일회성 알림을 등록하고, 화면 수명 주기에서 호출하는 해제 메서드는 추적 중인 같은 ID의 알림만 제거한다.

별칭 저장 성공 경로는 직접 `banner`를 대입하는 대신 `showTransientBanner`를 사용한다. 따라서 기존 4초 자동 해제와 포커스 해제 시 즉시 제거가 함께 적용된다.

## 화면 수명 주기

- `PopoverView`가 사라질 때 현재 일회성 알림을 제거한다.
- `SettingsView`가 사라지거나 앱의 scene phase가 비활성 상태로 바뀔 때 현재 일회성 알림을 제거한다.
- 제거 대상은 모델이 일회성으로 등록한 현재 알림뿐이다.
- 일회성 알림이 다른 오류나 경고로 교체된 경우, 이전 알림의 해제 요청은 새 알림을 제거하지 않는다.

## 테스트

1. 일회성 알림을 표시한 뒤 포커스 해제 메서드를 호출하면 알림이 제거되는지 확인한다.
2. 일반 오류 알림은 같은 메서드를 호출해도 유지되는지 확인한다.
3. 이전 일회성 알림이 새 알림으로 교체된 뒤에는 이전 해제 상태가 새 알림을 제거하지 않는지 확인한다.
4. 별칭 저장 성공 경로와 두 화면이 일회성 알림 수명 주기 API를 사용하는지 확인한다.
5. 전체 Swift 테스트와 빌드를 실행한다.

## 범위 밖

- 오류·경고 알림의 표시 정책 변경
- 계정 별칭 저장 로직이나 설정 파일 형식 변경
- 배포 또는 설치
